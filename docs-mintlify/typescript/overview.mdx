## ðŸ“¦ Modules

The source directory (`src`) provides numerous modules that one can use.

| Name                                 | Description                                                                                 |
| ------------------------------------ | ------------------------------------------------------------------------------------------- |
| [**agents**](/typescript/modules/agents.mdx)            | Base classes defining the common interface for agent.                                       |
| [**backend**](/typescript/modules/backend.mdx)          | Functionalities that relates to AI models (chat, embedding, image, tool calling, ...)       |
| [**template**](/typescript/modules/templates.mdx)       | Prompt Templating system based on `Mustache` with various improvements.                     |
| [**memory**](/typescript/modules/memory.mdx)            | Various types of memories to use with agent.                                                |
| [**tools**](/typescript/modules/tools.mdx)              | Tools that an agent can use.                                                                |
| [**cache**](/typescript/modules/cache.mdx)              | Preset of different caching approaches that can be used together with tools.                |
| [**errors**](/typescript/modules/errors.mdx)            | Base framework error classes used by each module.                                           |
| [**logger**](/typescript/modules/logger.mdx)            | Core component for logging all actions within the framework.                                |
| [**serializer**](/typescript/modules/serialization.mdx) | Core component for the ability to serialize/deserialize modules into the serialized format. |
| [**version**](/typescript/modules/version.mdx)          | Constants representing the framework (e.g., the latest version)                             |
| [**emitter**](/typescript/modules/emitter.mdx)          | Bringing visibility to the system by emitting events.                                       |
| **internals**                        | Modules used by other modules within the framework.                                         |
